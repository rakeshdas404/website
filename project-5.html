Optimizing Pathfinding in Pac-Man

Efficient Maze Navigation with Admissible and Consistent Heuristics

VIEW REPORT

Project Overview

This project focuses on solving the Pac-Man pathfinding problem by developing and evaluating an admissible heuristic for navigating mazes and collecting food pellets using the A search algorithm*. The primary objective was to optimize the performance of the foodHeuristic function to reduce the number of nodes expanded and computation time while maintaining the optimal path cost. The heuristic leverages the Manhattan Distance to estimate the shortest path from Pac-Man’s position to the nearest food pellet, ensuring admissibility and consistency. The heuristic design adheres to the triangle inequality property, which guarantees that it never overestimates the true cost, making it ideal for efficient search operations.

Performance evaluations were conducted on multiple maze layouts—tinySearch, trickySearch, and testSearch—demonstrating significant improvements in computation speed and node expansion rates. For instance, the heuristic reduced node expansions from 16,688 to 6,126 and computation time from 57.6 seconds to 2.9 seconds in the trickySearch layout, without affecting the optimal path cost. Similar results were observed across other layouts, validating the heuristic’s efficiency and correctness. The project not only highlights the importance of designing admissible and consistent heuristics but also provides insights into optimizing search-based AI algorithms for real-time decision-making in complex environments. key Files Modified: searchAgents.py – Custom heuristic implementation. search.py – Search algorithm configuration. util.py – Utility functions for optimization and debugging.
